Ⅰ. 개요
아이디어/서비스명
아이디어 / 서비스명
NotiSNU(노티스누)는 Notice(공지) + SNU(서울대학교)를 합성하여 만든 이름으로, 
여러 학부, 기관 등으로 흩어진 서울대 내의 공지들을 AI로 통합하고 개인 맞춤형으로 추천/알림을 제공하는
플랫폼입니다.

서비스 개요
:: 대학 내 공지사항은 단과대, 학과, 행정부서, 비교과 시스템, 메일 등 여러 채널로 흩어져 있으며, 정보
접근성의 불균형을 가지고 있습니다. 
:: ‘NotiSNU(노티스누)’는 이러한 정보 접근성의 불균형 문제를 해결하기 위해, 대학 내 흩어져 있는 각
종 공지사항을 한 곳에 통합하고, AI를 통해 개인의 관심사와 이력에 맞춰 공지를 추천해주는 플랫폼이
며, 다음과 같은 서비스를 제공합니다.
    대학 내/외부 공지 자동 수집 및 분류
    사용자의 관심사와 이력을 활용한 AI 기반 개인 맞춤형 공지 추천
    마감 임박 알림, 공지 요악, 개인 맞춤형 활동 검색을 위한 AI 챗봇 제공
이를 통해 “모르면 놓치는 기회”를 줄이면서 학생들의 정보 접근권을 실질적으로 보장하고, 또 사용자
는 본 서비스를 통해 본인에게 필요한 정보를 빠르게 확인할 수 있습니다.Ⅰ. 개요

Ⅱ. 아이디어 제안 배경
아이디어 제안 배경
1. 현재 대학 내 정보 전달 체계가 상당히 분산되어 있음
 :: 학부대학, 각 단과대학, 학과별 홈페이지, 비교과 시스템, 학교 메일 등 공지의 출처가 너무 많고, 통합된 관리 체계가 존재하
지 않습니다. 
2. 그 결과, 학생들은 자신에게 필요한 정보를 놓치는 경우가 많음
 :: 학생들은 자신에게 필요한 공지를 놓치지 않기 위해, 교내 여러 웹사이트나 메일 주기적으로 방문하고, 해당 내용들을 하나
씩 확인해야만 합니다. 이러한 어려움은 두가지 형태로 구체화됩니다.
2-1. 소속 단체 간의 공지 분산 (자신의 소속 단과대에는 있지만, 소속 학과에는 없는 공지)
 (실제 사례) 공과대학 소속의 학생이면 누구나 지원할 수 있는 ‘캐나다 Polytechnique Montr é al - 2026 Winter Research
Internship Program’은 기계공학부 홈페이지에는 공지로 올라오지 않고 공과대학의 홈페이지에만 올라와있습니다.
 :: 결과적으로, 학부 홈페이지만 확인하는 학생들은 해당 프로그램의 존재조차 알지 못한 채, 지원의 기회를 상실했습니다.
2-2. 비소속 단체의 공지 접근 제한 (지원 가능하지만, 정보 접근이 어려운 공지)
 (실제 사례) 2025년 초에 진행된  'LG AI 청소년 캠프 2기 멘토 모집' 공지의 경우, 기계공학부 학생도 지원 가능했지만, 공지는
전기정보공학부와 컴퓨터공학부 홈페이지에만 올라왔습니다.
:: 결국, 자신이 속하지 않은 학과 게시판을 우연히 방문하지 않는 이상 해당 기회를 알 방법이 없습니다. 
두가지 문제 모두 정보가 존재하지만, 접근 경로가 파편화 되어 있어, 개인이 이를 탐색하지 않으면 기회를 잃게 된다는 공통된 본질을 가지고 있습니다.
이는 대학 내 정보 전달 체계가 ‘정보 접근성의 차이’를 야기하는 구조적 문제를 내재하고 있다는 증거이기도 합니다.

3. 기존 비교과 관리 시스템 역시 형식적인 수준에 머물러 있으며, 사용자의 이력 기반 추천이나 공지 통합 기능은
존재하지 않음
 :: 비교과 시스템은 아직 일반 공지로 올라오는 다양한 활동들을 포함하고 있지 않으며, 개인 맞춤형 서비스 또한 제공
되지 않고 있습니다. 
4. 해당 문제는 공공 복지나 세금 환급과 같은 영역에서도 이미 확인된 바가 있음
 :: 이러한 정보 비대칭 문제는 대학 내에서만 존재하는 현상이 아니며, 공공 행정/복지/취업 등 다양한 영역에서도 유
사한 문제가 반복되어 왔습니다. 해당 문제의식에서 출발하여 플랫폼 형태의 서비스로, 사회적으로 긍정적 영향을 미친
사례가 존재하며, 이는 다음과 같습니다.
4-1. 세금 환급 영역 (‘삼쩜삼’)
 :: 많은 근로자들은 연말정산 이후에도 세금 환급을 추가로 받을 수 있음에도, 복잡한 세법 구조와 불명확한 안내로 인
해 그 사실을 알지 못했습니다. ‘삼쩜삼’은 세금 데이터를 자동으로 분석하여 환급 가능 금액을 간편하게 조회/청구 할
수 있도록 하였고, ‘몰라서 못 받던’ 환급금을 국민이 직접 돌려받을 수 있게 했습니다. 
4-2. 복지 영역 (‘웰로’) 
 :: ‘웰로’는 사용자의 나이, 소득, 건강 상태 등을 입력하면 실제로 받을 수 있는 복지 서비스를 추천해 주는 시스템으로,
일부 지자체별/기관별 복지 정보가 누락되어 자신이 받을 수 있는 복지를 ‘몰라서 신청하지 못하는’ 문제를 해결한 서비
스 입니다.
대학 내에서도 위 사례와 마차가지로, 학생들이 ‘몰라서 지원 못하는’ 구조를, AI 기반 통합 플랫폼을 통해 해소하고, 기회 접근성을 보장하는
서비스가 도입될 필요성이 있습니다.

2. 개인 맞춤형 AI 추천/요약 시스템
 :: 사용자의 관심사, 전공, 키워드 등을 기반으로 벡터화된 프로필을 생성합니다.
 :: 해당 프로필을 활용하여 사용자와 관련성이 높은 공지 내용을 LLM이 요악하고, 맞춤형 피드백 문장을 생성하여
사용자에게 제공합니다.
 :: 채팅을 통해 사용자의 구체적인 요구사항을 파악하고, DB에 등록된 데이터를 기반으로 하여,
관련 활동 및 공지를 제공해주는 AI 챗봇도 함께 운영하여, 동적인 정보 검색이 가능하도록 돕습니다.

Ⅲ. 아이디어 적용 및 확대 방안
구체적인 아이디어 구현, 실행방안, 정책 적용 가능성
아이디어 구현 및 실행방안
[PHASE 1]
 :: 공개된 공지 페이지를 우선으로 하여 크롤링 및 DB 구축을 진행합니다.
 :: 공지 자동 분류를 위한 로직을 개발하고, API 기반 분류 모델을 적용합니다.
 :: 키워드 기반으로 초기 개인화 추천시스템을 구축합니다.
 :: 사용자 친화적 웹 인터페이스를 구축하고, 관심사 기능을 구현합니다.
[PHASE 2]
 :: 로그인을 통해, 마이스누/비교과 시스템과 연동될 수 있도록 설계합니다.
 :: 시스템 연동을 통해 얻은 사용자의 지난 활동 이력, 전공, 관심사 데이터를
활용하여, AI 기반 추천 시스템을 구축합니다.
 :: LLM 모델을 사용하여 활동 추천/검색 AI 챗봇을 구축합니다. ‘마인드로직
인포미‘를 이용하여, 서울대학교 중앙도서관 웹서비스와 비슷하게 웹페이지 상
의 신뢰할 수 있는 정보들만을 제공할 수 있는 챗봇을 구축할 계획입니다.
 :: 알림 시스템 (메일, 카카오톡)을 구축합니다.Phase 1에서는 기본적인 프런트엔드와 백앤드 작업을 통
해, 본 플랫폼에서 제공하는 핵심 기능 구현을 위한 초기
작업이 진행됩니다.
Phase 2에서는 로그인을 통한 사용자 프로필 구축, 퍼스
널 데이터를 기반으로 한 AI 추천 시스템 및 챗봇 구축 작

Ⅲ. 아이디어 적용 및 확대 방안
확장성 및 기대효과
1. 교내 확장
 :: 비교과 관리 시스템 및 학생지원센터 등과의 연동을 통해,
모든 교내 활동 정보를 한눈에 볼 수 있는 통합 플랫폼으로 발전 가능합니다.
2. 교외 확장
 :: 외부 재단, 기업, 연구소 공지를 추가하여 “대학 정보 통합 플랫폼”으로 확장할 수 있습니다.
 :: “AI 기반 공지 게재 플랫폼”으로 발전하여 다양한 기관의 공지 홍보 채널로 사용될 수 있습니다.
3. 기대효과
학생
 :: 흩어진 정보를 한눈에 확인할 수 있도록 도와주어 정보 접근성을 높이고, 이는 다양한 활동의 참여 기회를 향상하
는데에 영향을 줄 수 있습니다.
대학
 :: 하나의 통합된 채널을 통해 중복된 공지를 줄이고, 이를 통하여 행정적 효율성을 높일 수 있습니다.
사회
 :: 실제로 생성형 AI를 활용하여 “정보 접근성 향상을 통해 학생 삶의 편의성을 높이는 사례”가 될 수 있습니다.
